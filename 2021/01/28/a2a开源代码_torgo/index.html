Preprocessing

TORGO文件结构

database按照说话人进行分类，F/M表示性别，C表示control，数字代表编号；然后每个说话人目录按照Session进行分类，session中包含了amp, pos, prompts, wav_arrayMic, wav_headMic等文件夹，当然并不是每一个Session都有，这代表我们需要一个脚本进行处理。

其中pos就包含了我们需要的articulatory相关数据，而wav_arrayMic和wav_headMic包含了需要的acoustic相关数据；之所以说“包含”是因为具体的特征是需要进行提取的。

更为具体的解释可以阅读TORGO数据库的官网的“TORGO Directory and file structure”内容，链接为http://www.cs.toronto.edu/~complingweb/data/TORGO/torgo.html

process_torgo_data

articulatory_inversion项目中对数据的目录结构为Raw_data/corpus/speaker/wav&ema&prompts；因此我们需要对下载下来的TORGO进行目录的重构，达到于项目要求类似。

由于TORGO中说话人有多个session，而每个session中的内容以四位数字进行编号，因此我们需要将session进行合并，并且将文件进行重命名，我们使用speaker_session_number.xxx的格式来命名，之所以要加上speaker，是因为在train的时候需要通过文件名对说话人进行定位。

同时，同一个session下，pos和wav中文件不一定一致，因此还需要对每个说话人的pos文件和wav文件进行一一校对。

具体代码可以参考链接

https://github.com/XiaoshanHsj/articulatory_inversion/blob/master/Preprocessing/process_torgo_database.py

model

网络结构是2个全连接层，2个双向LSTM层（300个神经元），一个全连接层（18个神经元）和一个卷积层；其中卷积层的作用是smooth。

对于每一个wav，网络的输入是（K，429）维度的声学特征，其中K是帧数，429是11个帧的上下文窗口的mfcc维度；网络的输出是（K，18）维度的articulatory trajectory。

同时每个LSTM层之后有一个BatchNorm层，其维度是hidden*2
