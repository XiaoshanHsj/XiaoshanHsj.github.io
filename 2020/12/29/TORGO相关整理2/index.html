Staged Knowledge Distillation for End-to-End Dysarthric Speech Recognition and Speech Attribute Transcription

​ 这篇论文中涉及到了ASR和ASAT(Automatic Speech Attribute Transcription)，下面只整理ASR部分。

Database

pre-train: 500-hour normal speech from the Librispeech corpus.

retraining: 2 hours of dysarthric speech (2 males and 1 females) and 4 hours of normal speech (4 males) from the TORGO database.

test: another 1 hour of dysarthric speech (2 males and 2 females) from the TORGO database.

data augmentation: other 100-hour normal speech from the Librispeech corpus.

Features

120-dim log Mel-filterbank energy features (40-dim static, Δ, ΔΔ)

each feature was mean- and variance-normalized; every four frames were spliced (three left, one current and zero right)

frequency: 20 Hz – 8,000 Hz

speech perturbation: 0.9, 1.0 and 1.1

System’s Architectures

Speech Transformer (seq2seq attention-based model) + teacher-student learning/knowledge distillation.

the teacher network is trained by optimizing the loss between the ground truth Y and the output softmax of the teacher Ot

the student network is trained to learn from selected labels (Y + the output softmax of Ot). The loss function is between the selected labels and outputs softmax of the student network Os.

To make a full use of limited data resources, the student model is first adapted to the mixture of multiple dysarthric and common speakers, and then adapted for the target dysarthric speakers.

In the first stage (acc ≤ λ), the selected labels : Y + Ot, In the second stage, the selected labels : Y + OS1.

Methods Description
S1 The full net is fine-tuned(ft-full)
S2 S1 + 100h-Libri data augmentation
S3 Only the decoder is fine-tuned (ft-decoder)
S4 The model is refactored. The layers in the decoder are shared (the parameters are fine-tuned by using all speech data from the TORGO database with speech perturbation)
TS1 The conditional teacher-student learning. The teacher model: S2 or S4; the student model is a refactored model trained by using TORGO with speech perturbation
TS2 The tunable tradeoff parameter λ is set to 0.95

Database

on sentence subset of TORGO

apply an N-fold cross-training setup with the total dataset being divided into five folds (i.e. one fifth of each speaker in every fold)

Features

Training data was augmented using speed perturbation (using factors 0.9, 1.0 and 1.1)

39-dim MFCC

40-dim fMLLR transformed features with splicing of 11 contextual frames

Background

Autoencoder-based bottleneck feature extractor

An AE is an unsupervised way to learn a compact data representation. (encoder + decoder)

Encoder: high dimensional input –> lower dimensional latent variable (AE-BN feature)

Decoder: reconstruct the input using latent variable

AE-BN: i) capture as much as the input data characteristics as possible

​ ii) discard the redundant information

Acoustic model architecture

Light Gated Recurrent Units (LiGRU) model has the capability to exploit large time context and to capture long-term speech modulations.

Compared with the LSTMs, LiGRUs have a simpler cell design that allows for faster training. The design also avoids the numerical issue of learning long-term dependencies and mitigates the vanishing gradient problem by employing ReLU activation with batch normalisation.

Multi-task optimisation

Joint optimisation

jointly optimising the AE-BN feature extractor and the speech recogniser where the feature extractor is pretrained on LibriSpeech data and fine-tuned using TORGO dysarthric data.

In this case, the speech recognition gradient is also backpropagated through the feature extraction module.

Monophone regularisation

the two tasks are jointly estimated by using a weighted sum cost function between the two predictions from the two softmax classifiers (predicting CD or CI)

This multi-task learning scheme can be regarded as a technique to regularise the AM

System’s Architectures

AE-BN feature extractor is trained on the 100-hour subset of LibriSpeech corpus

the loss function of feature extractor is minimising the mean square error between the reconstructed input data and the true input data

the loss function of classifier: cross-entropy loss between the CD/CI labels and the predictions
$$
Loss_{ASR} = Loss_{CD} + \lambda_{2} * Loss_{CI} , \
Loss_{Joint} = \lambda_{1} * Loss_{AE} + Loss_{ASR}
$$

four layers: 2 layers convolution layers with filter length 3 and ReLU activation + 2 layers feed-forward ReLU layers with 768 units and 20 units

Decoder:

two ReLU layers fed by learned AE-BN features and aims to produce an output matching the 440-dim input

Acoustic model:

LiGRU-based, containing five stacked bidirectional LSTM layers and a final softmax classifier.

dropout (0.15)

others:

minibatch 128 16

learning rate 0.5 0.5

LM: 200k vocabulary size LibriSpeech trigram LM

Exploring Appropriate Acoustic And Language Modeling Choices For Continuous Dysarthric Speech Recognition

Challenges of using dysarthric speech corpora for continuous speech recognition

There is a large overlap between ang given speaker’s utterances and those seen in their training set as all speakers have had the same prompts and contributed very similar utterances.

So any LM trained on any speaker’s corresponding training part of the dataset will be highly tuned to the test set.

Language Modelling & Results

AMs:

GMM-HMM: triphone model with speaker adapted transformation

DNN-HMM: following [Automatic Speech Recognition with Deep Neural Networks for Impaired Speech] (第一篇)

Leave-one-speaker-out cross validation

TORGO LM

a trigram LM built on prompts of the training stage data in TROGO.

Task-specific TORGO LMs

isolated word: unigram LM, 615 unique words

sentence: trigram model, 313 to 354 sentence prompts, applying Witten-Bell discounting

Out-of-domain LibriSpeech LMs

build LibriSpeech unigram LMs over a range of vocabulary sizes: {2k, 5k, 10k, 15k, 20k, 25k, 30k, 35k, 40k, 45k, 50k, 100k, 150k, 200k}

trigram LMs is built by pruning the pre-trained official 3-gram LibriSpeech LM

The OOV words are converted to <UNK> tag in the unigram while N-grams containing OOV words are removed, then the model is re-normalized.

To achieve comparable OOV rates for the smaller vocabulary sized LM for both the word and the sentence tasks, vocabulary sizes starting from 0.1k are introduced for the sentence task.
